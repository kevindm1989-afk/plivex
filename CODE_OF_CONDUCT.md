# Code of Conduct

This is a personal project. There is no community Slack, Discord, mailing list, or other gathering place. The only sanctioned communication channels are GitHub Issues and Pull Requests on this repository.

## Conduct in any interaction on this repository

- Be respectful.
- Stay on topic.
- Don't harass, threaten, dox, or attack other commenters.
- Don't post material that is illegal in the maintainer's jurisdiction (Ontario, Canada).
- Don't post spam, advertising, or off-topic promotional content.
- Don't impersonate the maintainer, other contributors, or other people.

## Maintainer's discretion

The maintainer reserves the right at their sole discretion, with or without notice or explanation, to:

- Lock or close any issue or pull request.
- Hide or delete any comment.
- Block any user from interacting with the repository.
- Limit, throttle, or disable repository features (issues, discussions, wiki).
- Archive the repository, ending all interaction.

There is no appeal process. There is no community vote. This is one person's project, published as open source, and the maintainer's decisions about how it is maintained are final.

## If this isn't for you

If you find this stance unwelcoming or want a more participatory project, the appropriate response is to fork. Plivex is MIT-licensed; you may run, modify, and redistribute your own copy under your own community norms. Forking relieves both you and the original maintainer of any expectation of community.

## Reporting violations

Conduct violations on this repository may be reported to the maintainer via GitHub's "Report" feature on the offending comment or user. The maintainer is not obligated to respond to reports outside of GitHub's tooling.

## What this document is not

This is not a contract. It does not create a legal obligation on the maintainer. It describes how the maintainer intends to operate the project. The software remains provided "as-is" per the MIT license; see `TERMS.md` for the full disclaimer.
